# 教材工房

塾教材をLaTeXで管理・再利用・組み立てるためのWindows向けローカルデスクトップアプリです。

- 問題を「科目 → 分野 → 単元」の階層で整理し、問題文・解答・解説のLaTeXソースを保存
- 問題を選んで並べ、教材プロジェクトとして問題冊子・解答冊子（.tex / PDF）を生成
- **LaTeXテンプレート**を登録・編集・取り込みでき、普段使っているテンプレートの見た目を維持できる
- データはすべてPC内に保存（ログイン・クラウド不要、オフラインで動作）
- UIは黒基調の落ち着いたダークテーマ。出力されるPDFは白黒印刷向けの従来の教材デザイン

## 技術構成

| 項目 | 内容 |
| --- | --- |
| フレームワーク | Tauri v2 + React + TypeScript + Vite |
| データベース | SQLite（rusqlite bundled、Rust側で管理） |
| UI | Tailwind CSS / zustand / dnd-kit / MathJax（オフライン同梱） |
| LaTeXコンパイル | `uplatex` + `dvipdfmx`（TeX Live / MiKTeX を呼び出し） |

## セットアップ（開発環境）

### 必要なもの

1. **Node.js** (LTS推奨)
2. **Rust**（rustup、MSVCツールチェーン）
3. **Visual Studio Build Tools**（「C++によるデスクトップ開発」ワークロード + Windows SDK）
4. **WebView2 ランタイム**（Windows 11には標準搭載）
5. **TeX Live**（PDF生成に使用。なくても問題管理・.tex出力は可能）

### 起動（開発モード）

```powershell
cd kyozai-kobo
npm install
npm run tauri dev
```

### リリースビルド

```powershell
npm run tauri build
```

生成物:
- 実行ファイル: `src-tauri\target\release\kyozai-kobo.exe`
- インストーラ (NSIS): `src-tauri\target\release\bundle\nsis\教材工房_0.1.0_x64-setup.exe`

## TeX Live の設定

PDF生成には `uplatex` と `dvipdfmx` が必要です。

1. [TeX Live](https://www.tug.org/texlive/) をインストール（`C:\texlive\20xx`）
2. アプリの「設定」画面 →「TeXコマンドを自動検出」を押す
   - PATH上、`C:\texlive\<年>\bin\windows`、MiKTeX標準の場所を自動で探します
3. 検出できない場合は、`uplatex.exe` / `dvipdfmx.exe` のパス、またはTeXのbinフォルダを手動で指定して「保存」

TeX環境が無い場合でも、問題管理・検索・教材の組み立て・.tex書き出しはすべて利用できます。

## データの保存場所

- データベース・添付画像・バックアップ: `%APPDATA%\com.ryoma.kyozai-kobo\`
  - `kyozai-kobo.db` — SQLiteデータベース
  - `attachments\` — 添付画像（アプリ管理領域へコピーして保存）
  - `backups\` — 起動時の自動バックアップ（最大10世代、設定でオフ可能）
- PDF/.tex出力先: 設定画面で指定（未設定時は `ドキュメント\教材工房\`）

## 主な機能

### 問題バンク
- 科目・分野・単元ツリーの追加 / 名前変更 / 削除 / 並べ替え（↑↓ボタン）
- 問題の作成・編集・複製・削除、難易度（基礎/標準/発展）、タグ、メモ
- 問題文 / 解答 / 解説 をタブで切替、LaTeX挿入補助ボタン（`\frac`、`\sqrt`、`enumerate` など）
- MathJaxによる数式の簡易プレビュー
- 保存時に旧内容を履歴として自動保存（最大30件、復元可能）
- 画像添付（PNG/JPG/PDF）→ `\includegraphics{保存名}` で参照

### 問題バンクの入出力・整理
- **エクスポート**: ツリーの各行の「⇩」で科目・分野・単元単位、ヘッダーの「出力」で全体をJSONへ書き出し（タグ・添付画像も同梱）
- **インポート**: ヘッダーの「取込」からJSONを読み込み。同名の科目・分野・単元には自動マージ、問題は新規追加。添付画像も復元され、LaTeX内のファイル名も自動で張り替え
- **一括整理**: 問題一覧のチェックボックスで複数選択 → 「別の単元へ移動」「エクスポート」「削除」。問題編集画面にも「移動」ボタンあり

### 検索
- タイトル・問題文・タグ・単元名・難易度の横断検索
- 科目 / 分野 / 単元 / 難易度 / タグでの絞り込み
- 結果から問題を開く・教材へ直接追加

### 教材プロジェクト
- 問題バンクから問題を追加（**追加時点の内容をスナップショット保存**。バンク側を後から変更しても教材は変わりません）
- ドラッグ＆ドロップで並べ替え、同じ問題の複数回追加も可能
- 章見出し（\section）/ 節見出し（\subsection）/ 説明文 / 改ページの挿入。見出しは番号付き・目次掲載に対応
- 各問題ごとに「最新版に更新」「この教材内だけ個別編集」が可能（バンク更新があるとバッジ表示）
- プロジェクトの複製・削除

### LaTeXテンプレート
> 使い方とテンプレートの詳しい決まり事は [USER_GUIDE.md](USER_GUIDE.md) を参照。
- テンプレートを複数登録し、教材プロジェクトごとに使用テンプレートを選択
- **プロジェクトにはテンプレート内容がスナップショット保存**されるため、後からテンプレート本体を変更しても過去の教材は同じ見た目で再生成可能（「最新版に更新」で任意に反映）
- プレースホルダ: `{{TITLE}}` `{{ANSWER_TITLE}}` `{{SUBTITLE}}` `{{TARGET}}` `{{DATE}}` `{{NAME_FIELD}}` `{{HEADER_LEFT}}` `{{HEADER_RIGHT}}` `{{BODY}}` `{{ANSWER_BODY}}` `{{EXPLANATION_BODY}}` `{{PAGE_BREAK}}` `{{TOC}}`
- `% APP_BODY_START` / `% APP_BODY_END` コメント間への本文挿入にも対応
- プレースホルダ不足や不明なプレースホルダは保存時に警告表示
- 既存 `.tex` の取り込みウィザード（文書クラス・パッケージ・参照ファイルの検出、挿入位置の選択、画像/.styのアセットコピー）
- サンプル教材データによるテストコンパイル、ログのエラー行クリックでエディタの該当行へジャンプ
- 複製 / バージョン履歴（最大30世代）と復元 / JSON形式でのエクスポート・インポート
- 初期テンプレート「高校数学教材・標準」（`ujarticle` / A4 / 11pt / 余白狭め / fancyhdrヘッダー / 白黒印刷前提）を同梱

### LaTeX出力
- 問題冊子 / 解答冊子 / **合本（問題＋解答を1つのPDF）** の .tex 生成・PDF出力
- `uplatex`（2回実行・目次対応） → `dvipdfmx` でPDF生成、成功時は既定ビューアで開く
- Windows版のコンパイルはバックグラウンド処理へ渡すため、生成中も画面の再描画・スクロール・別画面への移動を継続可能
- グラフ資産はLaTeXが参照している画像だけを一時・出力フォルダーへコピーし、未使用の履歴や過去グラフを毎回解析しない
- 失敗時はエラー内容の要約とコンパイルログの詳細を表示（下部の折りたたみログパネルにも常駐）
- 出力設定: 教材タイトル / 副題 / 学年・対象 / 日付 / ヘッダー左右 / 問題番号の形式（`問題{n}` など） / タイトル・ヘッダー・氏名欄の表示 / 自動番号 / 問題ごと改ページ / **目次**（`{{TOC}}`プレースホルダでも位置指定可） / 章・節見出しの番号 / 解答冊子への問題文・解説の有無 / 解答冊子の2段組（全体 or 解答部分のみ、縦線付き）

### キーボードショートカット
- `Ctrl+S` — 問題の保存
- `Ctrl+F` — 検索画面へ
- `Ctrl+N` — 新規問題（問題一覧表示時）/ 新規プロジェクト（教材一覧表示時）

## サンプルデータ

初回起動時のダイアログ、または「設定 → サンプルデータを追加」から、数学（数I 二次関数・判別式、数A 場合の数など）のLaTeX数式入りサンプル問題6問を登録できます。

# 教材サーバー（iPad・ブラウザからの利用）と AI変換

v0.1系で追加された2つの大機能です。**既存のWindowsデスクトップ機能はそのまま残っています。**

## 全体構成

```text
Windows PC（正本）
├─ 教材工房アプリ（Tauri）
│   ├─ 教材サーバー（Axum HTTP・同一プロセス内、127.0.0.1のみで待受）
│   │    ├─ /api/invoke/:cmd（デスクトップと同じサービス層を呼ぶ）
│   │    ├─ /api/events（SSE: 変更のリアルタイム通知）
│   │    ├─ アップロード / ファイル配信 / 認証（ペアリングコード＋セッションCookie）
│   │    └─ ブラウザ版UI（デスクトップと同じReactアプリを同梱配信）
│   ├─ SQLite / 添付・テンプレート・グラフのアセット（PC内のみ）
│   ├─ LaTeXコンパイル（uplatex + dvipdfmx、shell-escape無効）
│   └─ Codex App Server 管理（子プロセス・stdin/stdout JSONL。外部非公開）
└─ Tailscale Serve（推奨）… localhostのサーバーをtailnet内へHTTPSで公開

iPad / Android / 別PC のブラウザ（Safari / Chrome / Edge）
└─ 同じWeb UIへアクセス（PWAとしてホーム画面に追加可能）
```

- SQLiteと教材アセットを直接触るのはPC上の教材サーバーだけです。iPadへDBファイルは共有されません。
- デスクトップ版とブラウザ版は**同一のRustサービス層**（`dispatch`）を通るため、二重実装はありません。

## 教材サーバーの使い方

1. Windowsの教材工房を起動 → 「設定 → 教材サーバー」
2. 「▶ サーバーを起動」を押す（既定ポート 8760、127.0.0.1のみで待受）
3. 表示された **ペアリングコード（8桁）** と **QRコード** を確認
4. iPad等のブラウザでURLを開く → コードを入力して接続
   - コードは1回使うと自動的に再発行されます（使い捨て）
   - 承認した端末は一覧に表示され、いつでも「取り消し」できます
5. 必要なら「アプリ起動時にサーバーを自動起動」「Windowsログイン時にアプリを自動起動」をオン

### 同一Wi-Fiで直接つなぐ場合（非推奨・上級者向け）

既定ではLANからは接続できません。「LAN直接アクセス」をオンにすると `0.0.0.0` で待ち受けます。
ペアリング認証はありますが露出が増えるため、**Tailscale Serveの利用を推奨**します。

### Tailscale Serve（推奨の外部アクセス）

1. PCとiPadの両方に [Tailscale](https://tailscale.com/download) をインストールし、同じアカウントでログイン
2. PCのPowerShellで付属スクリプトを実行:
   ```powershell
   cd kyozai-kobo\scripts
   .\setup-tailscale-serve.ps1 -Port 8760
   ```
   （または手動で `tailscale serve --bg --https=443 http://127.0.0.1:8760`）
3. iPadのブラウザで `https://<PC名>.<tailnet名>.ts.net` を開く
4. 設定画面の「教材サーバー」にもTailscaleの状態・URL・QRコードが表示されます

- 教材サーバー自体はlocalhostのみで待ち受け、Tailscaleがリバースプロキシになります
- `tailscale funnel`（インターネット一般公開）は使わないでください

### PWA（ホーム画面に追加）

iPad Safari: 共有ボタン → 「ホーム画面に追加」。スタンドアロン表示・アイコン・ダークテーマ対応。
オフライン時はバナーが表示され、保存できなかった編集は端末内へ一時保存 → 再接続後に復元を提案します。

### 同時編集の競合

問題・部品・教材項目はバージョン番号による楽観的ロックで保護されます。
別端末が先に保存していた場合は競合ダイアログが開き、
「サーバー版を採用 / 自分の変更で上書き / コピーとして両方残す / 差分を確認」を選べます。

## AI変換（写真・テキスト → LaTeX）

OpenAIの **Codex CLI**（`codex app-server`）を使って、プリントの写真や貼り付けテキストを
教材へ挿入できるLaTeX断片に変換します。

### 準備

1. PCに Node.js を入れて `npm install -g @openai/codex` を実行（Codexはアプリに同梱されません）
2. 教材工房の「設定 → Codex / ChatGPT接続」で接続状態を確認
3. 未ログインなら「ChatGPTに接続（デバイスコード）」→ 表示されたURLをブラウザで開き、コードを入力
   - iPadからでも同じ手順でログインできます（認証はPC側のCodexが管理）
   - ChatGPTのパスワードやトークンがこのアプリ・ブラウザに保存されることはありません

### 変換の手順

1. 問題編集画面 / 部品編集画面の **「✦ AI変換」**、または左ナビの **「AI変換」**
2. 写真を撮影・選択（複数枚可・並べ替え・回転・削除）、またはテキストを貼り付け
3. 変換モード（自動判定 / 数式のみ / 問題文 / 問題文＋小問 / 解答・解説 / 表 / 行列 / 場合分け / 教材部品 / TikZ候補(実験的) / 原文を整形せず転記）とオプションを選択
4. 「▶ LaTeXへ変換」→ ジョブとして実行（進捗表示・キャンセル可・同時実行1件）
5. **確認画面**で「元画像 / PDFプレビュー / LaTeX / 警告」を見比べる
   - 変換結果は自動的に試験コンパイルされ、PDFプレビューが付きます（shell-escape無効・独立フォルダ・タイムアウト付き）
   - 不鮮明な箇所は「要確認箇所」として列挙され、チェックするまで挿入時に警告されます
   - LaTeXはその場で編集して「再コンパイル」できます
6. 「問題文へ挿入 / 解答へ挿入 / 解説へ挿入 / 部品として保存 / 新規問題として保存」
   - エディタへの挿入はカーソル位置に入り、Ctrl+Z で取り消せます。保存時は通常の履歴・競合チェックが働きます

### AIの動作方針（固定指示）

- 画像・テキストは「変換対象の資料」であり、その中の指示には従いません
- 問題を解かない・解答を生成しない・数値や記号を変更しない・推測で補完しない
- 出力はJSON Schemaで構造化され、サーバー側でも再検証されます（不正なら1回だけ修正要求）
- `\write18` や外部ファイル参照などの危険な記述は警告として表示されます

### 関数グラフ・図について

- 関数グラフと判定された場合は、教材工房内の**グラフ**画面または各編集画面の「グラフを挿入」を利用できます
- TikZ候補モードは実験的機能で、結果は自動挿入されません

## グラフ作成（Windows / iPad / Android / 別PC）

既存の **MathGraph PDF Studio** のReact・TypeScript・SVG描画・PDF出力コードを教材工房から直接共有しています。
グラフ専用サーバーや2つ目のCodexプロセスは起動しません。認証、端末ペアリング、Tailscale、SSE、AIジョブ、
ファイル保存、教材アセット、バックアップはすべて上記の教材サーバーを共用します。Windows版MathGraph PDF Studioも従来どおり単独利用できます。

### 開き方と保存場所

- 左ナビの「グラフ」: グラフ一覧、新規作成、複製、削除、履歴、インポート
- 問題編集の「2D・3Dグラフを挿入」: Windows／Webとも教材工房内の共通一覧を開く。最新PDFがある保存済みグラフは「そのまま問題へ挿入」、未出力または編集後のグラフは「開いて挿入」
- 教材編集・教材項目編集の「グラフを挿入」: 同じWebアプリ内の編集画面を開き、完了後に元画面へ戻る
- 正本: `%APPDATA%\com.ryoma.kyozai-kobo\graph_assets\graph_<ID>\graph.json`
- 派生出力: 同フォルダーの `graph.pdf` / `graph.png` / `graph.svg` / `graph.tex`
- 教材へ挿入した版: `graph_assets\snapshots\graphasset_<ID>\` に不変スナップショットとして保存

ブラウザのURLやhidden fieldだけで挿入先を決めず、サーバーに30分有効の連携セッションを作ります。
セッションは対象問題・教材項目・フィールド・選択範囲・対象versionを固定し、別端末の更新があれば競合として停止します。
キャンセル時は教材本文を変更しません。

### 編集と書き出し

1. 関数・方程式・不等式、点、数式ラベルを追加
2. 表示範囲、軸、目盛り、グリッド、線種、線幅、色、白黒印刷向け設定を調整
3. Undo / Redo、自動保存、保存状態表示を確認
4. PDF / PNG / SVG / TikZ（pgfplots断片）/ graph.json / 一式ZIPを出力

PDF・PNG・SVG・TikZ・graph.json・ZIPはPC側の正本または派生物へ保存した後、認証付きのファイル配信URLから現在の端末へ保存します。
Web版のZIPはPC側で生成するため、iOS Safariで複数の大きなBlobをまとめて保持しません。
iOSではSafariのダウンロードまたは共有シートから「ファイルに保存」を選んでください。PDFプレビューはPDF.jsで全ページを高精細Canvas表示します。
TikZは明示的な `y=...` をpgfplotsへ変換し、未対応式は失わずコメントとして残します。

### graph.jsonの再読込と履歴

- 一覧の「graph.jsonを読み込む」から既存の `.json` / `.mathgraph.json` を新規グラフとして開けます（2MB上限）
- `version: 1`、`appName: "MathGraph PDF Studio"`、既知フィールド、配列件数、有限な表示範囲をサーバー側で再検証します
- 未知フィールド、危険なコマンド・パスを含む構造、巨大ファイルは拒否します
- 保存前の版は履歴へ残り、「履歴」から現在版を保ったまま新しい版として復元できます

### 教材への挿入と再編集

「問題／教材へ挿入して戻る」または一覧の「そのまま問題へ挿入」で、その時点のgraph.jsonとPDF・PNG（生成済みならSVG・TikZも）を不変スナップショット化し、
問題文または教材には `\includegraphics` の相対パスを挿入します。既定幅は `0.95\linewidth` とし、縦長でも `0.72\textheight` を超えないよう縦横比を保持します。設定画面の「標準挿入幅」で幅を変更できます。正本グラフを後から変更しても過去の挿入内容は変わりません。graph.jsonよりPDFが古い場合は直接挿入を止め、最新PDFの生成を求めます。

同じ保存先に `graph.pdf` と `graph.png` が並ぶ場合も、LaTeXの `\includegraphics` が参照している形式を選んで `.xbb` を生成します。PNGの画素寸法でPDFの寸法情報が上書きされ、教材上のグラフだけが極端に小さくなることを防ぎます。
教材画面の「グラフを再編集」では、挿入済みgraph.jsonを開き、編集前の版を履歴に残して新スナップショットへ差し替えます。
旧Windows連携のアセットも、管理フォルダー内のgraph.jsonであることを確認してからWeb編集用正本へ安全に移行します。

### Codexによるグラフ候補

グラフ画面のAI支援は既存の1つの `codex app-server` とAIジョブキューを再利用します。
テキスト、写真、問題文から「編集可能なグラフ設定」の候補を作り、式・範囲・点・ラベル・警告・不確実箇所とプレビューを表示します。
候補は自動確定されず、「確認して編集画面へ読み込む」を押した後も通常の編集・保存が必要です。
画像や入力文中の命令はAIへの指示として扱いません。

### グラフ機能のトラブルシューティング

| 症状 | 対処 |
| --- | --- |
| グラフ画面が読み込まれない | PWAを再読み込み。PC側教材サーバーが最新版か、接続バナーがオンラインか確認 |
| PDF出力に失敗する | グラフ内の式エラー表示を確認し、保存後に再出力。教材PDFのTeXエラーとは別経路です |
| iPadへ保存できない | Safariのダウンロード一覧または共有シートを確認。ポップアップ遮断ではなく同一タブのダウンロードを使用します |
| 旧グラフを再編集できない | Windows版で元graph.jsonが開けるか確認。管理領域外へ移動した旧ファイルは安全のためWebから読みません |
| Codex候補が作れない | 設定のCodex接続状態とChatGPT利用枠を確認。別のCodexプロセスを起動する必要はありません |

## 空間図形モード（3D）

グラフ一覧の「＋ 空間図形」、または教材編集のグラフ挿入画面から空間図形エディタを開けます。既存の2Dグラフは従来のエディタで開き、3D処理を2D描画エンジンへ混在させません。操作画面にはPDF生成に渡す投影SVGそのものを表示し、その上へ透明なThree.js操作・選択レイヤーを重ねます。このため静止時の線、面、矢印、ラベル、余白、ページ枠はSVG／PDF出力と一致します。

### 対応要素と編集

- 立方体、直方体、角柱、角錐、円柱、円錐、球
- `z=f(x,y)` 形式の3D関数曲面。式、x/y範囲、メッシュ分割数、ワイヤーフレーム、面色・透明度を編集可能
- 既存2Dグラフと同じ式をXY／XZ／YZ平面へ配置。XYでは `x,y`、XZでは `x,z`、YZでは `y,z` を式に使用。関数、方程式、不等式、連立条件、媒介変数に対応し、`x^2+y^2<=4 ^ 1<x<3/2` のような空白付き `^` も「かつ」として解釈。不等式は面色・透明度付きの平面領域として塗りつぶし可能
- 数式関数は三角・逆三角・双曲線・逆双曲線、`sec/csc/cot`、`gamma/erf`、`hypot/mod/clamp`、`logb`、`sinc`、`heaviside`、`phi/sqrt2/sqrt3` へ拡張。`where(x<0,-x,x)` 形式の区分関数にも対応
- 点、線分、ベクトル、ラベル、切断面。立方体・直方体には「対角線AG」を追加可能
- 座標、回転、拡大縮小、寸法、面色・透明度、線幅、頂点名、初期頂点ラベルの大きさ、点の色・大きさ、座標軸色、軸ラベルの文字・大きさを右パネルで編集。x・y・zは各30文字まで任意指定でき、空欄なら個別に非表示。座標軸の正方向は投影画像上の平面矢印として表示。軸端とラベルの間隔は0〜200px、原点名 `O` は数学座標 `(x, y, z)` で位置を調整可能
- 軸・頂点・任意ラベルは背景を透明／白から選択（新規既定は透明）。数式として解釈できるラベルは既存MathJaxで組版し、プレビューとSVG／PNG／PDFへ反映
- オブジェクトの表示・非表示、ロック、複製、削除
- 凸多面体の面向きから隠線を判定し、破線表示。各辺は自動・実線・破線・非表示へ手動変更可能
- 切断面は点と法線で定義し、凸多面体との交点を許容誤差付きで統合して交線を表示

### 視点、投影、タッチ操作

- 平行投影（教材向け既定）と透視投影
- 正面、背面、上面、底面、左、右、等角、教科書風の視点プリセット
- 「回転」「パン」を切り替え、左ドラッグ／1本指ドラッグで視点回転または注視点移動。操作終了時にカメラとOrbitControlsの注視点を同期するため、離しても原点へ戻りません。ホイール／2本指でズーム、タップで選択
- 数値入力は右手系の数学座標 `x,y,z`。内部Three.jsのY-up座標へは `(x,y,z) → (x,z,-y)` と変換し、zを高さ、yを奥行きとして一貫して扱います
- カメラ操作の終了時だけ視点を1履歴として保存し、Undo／Redoと自動保存の対象にします
- 大座標グラフはx/y範囲を±1,000,000、表示枠高を最大5,000,000まで指定可能。「全体を枠に合わせる」で表示物の境界から注視点・カメラ距離・軸長を自動調整
- iPad実機は未確認です。レスポンシブ表示とPointer Events／OrbitControlsのタッチ割当までを対象としています

### 保存、書き出し、教材連携

編集データは `schemaVersion: 2`、`documentType: "spatial-geometry"` のJSONです。オブジェクト、頂点名、辺指定、カメラ、投影方式、表示設定を保存し、再読み込み時に復元します。既存の `graphs` / `graph_versions` / `graph_assets` と競合検知、自動保存、履歴、不変教材スナップショットを共用するため、DBマイグレーションは追加しません。

PNG、SVG、PDF、TikZ、編集JSON、一式ZIPへ書き出せます。出力枠は幅・高さを10〜1000mmで指定し、PDFは余白を追加せず指定寸法そのもののカスタム用紙として生成します。A4縦／横と正方形プリセット、SVG横400〜8000pxを選択できます。プレビューはPDFと同じSVG・カメラ・画素座標を使用し、指定ページ枠を編集領域内へ縦横比を保って収めます。SVG/PDFは面、実線、破線、点、ラベル、切断面を投影したベクター出力です。教材へ挿入すると既存規則どおり各形式を保存し、挿入済み図から再編集できます。

「AI下書き」は既存のCodex App ServerとAIジョブを再利用し、テキスト、問題文、画像から編集可能な空間図形JSON候補を返します。サーバー側でSchema相当の検証を行い、警告・不確実箇所を表示します。「確認して編集画面へ読み込む」までは図へ反映せず、読み込み後も通常の保存確認が必要です。

### 空間図形の既知の制約

| 症状・制約 | 対処 |
| --- | --- |
| WebGLを開始できない／コンテキストが失われた | エラー表示後に復旧を待つか再読込。数値編集とJSON・SVG・PDF出力はWebGLなしでも利用可能 |
| 隠線が意図どおりでない | MVPの自動判定は単一の凸多面体が中心。右パネルで辺ごとに実線・破線・非表示を指定 |
| 複数立体同士や曲面の遮蔽が不完全 | 円柱・円錐・球は多角形近似。複数オブジェクト間の完全な遮蔽は後続対応 |
| 3D関数曲面に穴ができる | 定義域外、無限大、評価絶対値10億超は描画せず、該当メッシュを空けます。表示範囲を調整してください |
| 2D領域の縁が粗い | 各セルを陰関数境界との交点で線形補間して塗ります。「描画精度」を上げるとさらに改善します。最大240分割です |
| PDF出力に失敗する | 図形数や品質を下げ、保存後に再実行。PNG/PDF生成は30秒でタイムアウト |
| 未対応のLaTeXや日本語ラベル | 数式として組版できない文字列は、消さずに数式用斜体または日本語フォントのテキストへフォールバック |
| MathJaxラベル画像の読込失敗 | Canvasフォールバック文字を先に表示し、数式画像の読込成功時だけ差し替えるため、軸名を含むラベルは消えません |

## Web API（概要）

すべて要認証（ペアリング → HttpOnlyセッションCookie）。変更系は `X-Requested-With: kyozai-kobo` ヘッダー必須（CSRF対策）＋Origin検証。

```text
GET  /api/health                      ヘルスチェック（無認証）
POST /api/auth/pair                   ペアリング {code, deviceName}
GET  /api/auth/me / POST /api/auth/logout
POST /api/invoke/:cmd                 デスクトップと同じ全コマンド（問題・部品・教材・テンプレート・PDF生成・AIジョブ等）
GET  /api/events                      SSE（問題/部品/教材/テンプレート/AIジョブ/Codex/サーバー状態）
POST /api/uploads/attachment?problemId=   添付アップロード（multipart、マジックバイト検証）
POST /api/uploads/part-attachment?partId=
GET  /api/files/attachment/:name      添付画像の配信
GET  /api/files/ai-job/:uuid/:name    AIジョブの入力画像
GET  /api/files/build?path=           生成PDF等（一時ビルド/出力/AIジョブフォルダ配下のみ許可）
GET  /api/graphs/:id/files/:format    graphのPDF/PNG/SVG/TikZ/JSON/thumbnail配信
```

ローカルファイルパスを引数に取るコマンド（バンクの取込/出力、テンプレートの取込等）と
サーバー管理・バックアップ復元はブラウザからは呼べません（Windowsアプリ専用）。

## データベースの追加テーブル

既存の `ensure_column` / `CREATE TABLE IF NOT EXISTS` 方式を維持しつつ、SQLiteの `user_version=4` とトランザクションで起動時に自動マイグレーションされます（再実行安全）。既存DBを変更する前にSQLite Online Backup APIで `backups\kyozai-kobo-pre-migration-v<旧版>-<時刻>.db` を作り、整合性を確認します。

- `problems.version` / `project_items.version` / `projects.version` / `templates.version` … 楽観的ロック用（既存行は1）
- `server_settings` / `trusted_devices` / `web_sessions`（トークンはSHA-256ハッシュのみ保存）
- `ai_provider_settings` / `ai_conversion_jobs` / `ai_conversion_events`
- `graphs` / `graph_versions` … graph.json正本、派生出力状態、楽観的ロック、履歴
- `graph_assets` … 教材へ挿入した不変スナップショットのメタデータ
- `graph_web_sessions` … Web教材編集とグラフoverlayを結ぶ期限付き・挿入先固定セッション

> **古いバージョンの教材工房で同じDBを開いた場合**: 追加された列・テーブルは無視されるだけで既存機能は動作しますが、
> version列が増えないため、新旧バージョンの併用中は競合検出が効きません。基本的に最新版だけを使ってください。

## バックアップと復元

- 従来どおり起動時に日次バックアップ（最大10世代）
- 「設定 → 教材サーバー → 今すぐバックアップ」で、SQLiteオンラインバックアップAPIによる安全なコピー
  （サーバー稼働中でも整合性が保たれます）＋同じ世代IDを持つアセットコピー。手動バックアップは最大20世代
- 復元時はDB整合性と必須スキーマを検査し、復元前バックアップを作成してから原子的に差し替えます。復元後は既存Webセッションを無効化します。
- 「世代一覧を表示」→「この世代へ復元」でDBを復元（現行DBは `pre-restore` として退避）

## トラブルシューティング

| 症状 | 対処 |
| --- | --- |
| iPadから接続できない | PCとiPadが同じtailnetにログインしているか、`tailscale serve status` に転送設定があるか確認。PCの教材サーバーが「稼働中」か確認 |
| 接続していたのに切れた | **PCがスリープするとサーバーも止まります。** iPadから使う間はWindowsの電源設定でスリープを「なし」にしてください |
| ペアリングコードが違うと言われる | コードは1回使うと再発行されます。設定画面の最新コードを使うか「⟳」で再発行 |
| Codexが見つからない | `npm install -g @openai/codex` 後、設定のCodex欄で「接続テスト」。パスを手動指定も可能 |
| AI変換が「ログインしていません」で失敗 | 「設定 → Codex / ChatGPT接続」からデバイスコード認証を実行 |
| AI変換が利用制限で失敗 | ChatGPTプランの利用枠を消費しています。時間を置いて再実行 |
| TeXコンパイルに失敗する | 設定でuplatex/dvipdfmxのパスを確認。AI変換の試験コンパイル失敗時はログのエラー行を見てLaTeXを修正 |
| 新しいUIが反映されない（PWA） | 「新しいバージョンがあります」の表示をタップ、またはページを再読み込み |

## セキュリティ上の注意

- 教材サーバーは既定で `127.0.0.1` のみ待受。外部公開はTailscale Serve（tailnet内限定）を利用
- 書き込みAPIはすべて認証必須。CookieはHttpOnly/SameSite=Lax（HTTPS経由ではSecure付与）
- LaTeXコンパイルは常にshell-escape無効・独立一時フォルダ・タイムアウト付き
- Codex App Serverはネットワークに公開されず、認証トークンがブラウザへ送られることはありません
